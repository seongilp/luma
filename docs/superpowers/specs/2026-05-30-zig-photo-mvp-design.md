# zig_photo MVP 설계

**날짜:** 2026-05-30
**언어/스택:** Zig + raylib (macOS 네이티브 단일 실행파일)
**목적:** Zig 학습 + 실사용 가능한 사진 관리 앱. 성능 우선.

## 큰 그림

Apple Photos급 전체(라이브러리 인덱싱·메타데이터·검색·정리·편집)는 여러 독립 시스템의 합. 한 번에 가지 않고 **작동하는 코어부터** 쌓는다. 본 문서는 1단계(MVP)만 다룬다.

이후 단계(순서 미확정, 별도 스펙): EXIF/메타데이터 → 검색·분류 → 디스크 썸네일 캐시·SQLite 인덱스 → 정리(이름변경/이동/중복제거/태그) → HEIC 지원.

## MVP 범위

- 폴더 선택 → 이미지 파일 **재귀 스캔**
- **썸네일 그리드** (스크롤, GPU 렌더링)
- 썸네일 클릭 → **큰 이미지 뷰어** (좌우 이동)
- 포맷: **JPEG/PNG 우선** (HEIC 제외)
- 썸네일: **메모리(LRU) 캐시만** (디스크 캐시·DB는 이후 단계)

## 아키텍처

단일 네이티브 실행파일. raylib = 윈도우/입력/GPU 렌더. Zig = 스캔·이미지 로딩·썸네일·UI 상태.
파이프라인: 스캔 → (지연) 썸네일 생성 → GPU 텍스처 업로드 → 그리드 렌더.

## 모듈 (작은 파일, 단일 책임)

| 모듈 | 책임 |
|------|------|
| `main.zig` | 진입점, 앱 루프, 조립 |
| `scanner.zig` | 폴더 재귀 스캔 → 이미지 경로 목록 (확장자 필터) |
| `image_loader.zig` | 경로 → 디코딩 픽셀 (raylib LoadImage) |
| `thumbnail.zig` | 원본 → 축소 썸네일 + GPU 텍스처화 |
| `gallery.zig` | 그리드 레이아웃·스크롤·선택, 그리드 렌더 |
| `viewer.zig` | 단일 이미지 큰 보기, 좌우 이동 |
| `app_state.zig` | 전체 상태 (현재 폴더, 이미지 목록, 모드 grid/viewer) |

## 데이터 흐름

1. 시작 시 폴더 경로(인자/기본값) → `scanner` 경로 목록 반환
2. **화면에 보이는 썸네일만 지연 로딩** (수천 장 성능 핵심)
3. 텍스처 **LRU 메모리 캐시**, 화면 밖으로 충분히 나가면 언로드 (VRAM 보호)
4. 클릭 → `mode = viewer`, viewer가 원본 풀해상도 로드

## 핵심 결정

- **지연 로딩 + LRU 텍스처 캐시** (대량 이미지 성능의 핵심)
- **동시성은 메인 스레드로 시작** (YAGNI). 스크롤 끊김 보이면 그때 백그라운드 디코드 스레드 도입.

## 에러 처리

- 깨진/미지원 파일 → 그리드에 placeholder(회색 박스+파일명) + 로그. 크래시 금지.
- 폴더 접근 실패 → 사용자 메시지.

## 테스트

순수 로직은 `zig test` 단위테스트: `scanner`(필터·재귀), `thumbnail`(축소 비율), `gallery`(좌표↔인덱스), LRU 캐시. raylib 렌더는 수동 검증.

## 빌드

`build.zig`로 raylib 링크. macOS는 Cocoa/OpenGL 프레임워크 함께 링크.
선행: zig 설치, raylib 의존성 확보(zig package manager 또는 시스템 설치).
